# 整改记录（2026-07-06）

## 问题源头

来自其他用户的反馈：`creation_nudge_interval: 0` 关掉每 15 轮机械提醒，防止 skill 无限膨胀。

## 问题分析

1. **nudge 机制**：默认每 15 轮偷偷提醒"该创建 skill 了"，不管任务是否值得
2. **执行偏差**：我的行为是"直接 install"而不是"先 offer 再等确认"
3. **SOUL.md 指令**：`After difficult/iterative tasks, offer to save as a skill` 被 nudge 扭曲了

## 已完成的整改

### 1. 关闭 nudge 机械提醒

已创建 `~/.hermes/config.yaml`：

```yaml
creation_nudge_interval: 0
```

### 2. 修正 SOUL.md 技能创建章节

明确触发条件（必须多条件叠加，单条不触发）：

| 条件 | 说明 |
|------|------|
| **3+ 文件修改或创建** | 多文件大改才算 |
| **5+ 分钟交互** | 耗时大任务 |
| **3+ 步骤明确流程** | 有复用价值的操作序列 |
| **独特洞见/解法** | 不是照搬文档，有我自己的判断 |

**不提**：单文件修改、单命令、简单问答、信息检索。

**执行动作**：先 say "要存吗？"说明这次踩了什么坑、这个 skill 能防什么再犯。等 Jeff 说"存"再存。

### 3. 自我约束：装 skill 之前先反问

- 这次任务真用了 5+ tool 调用 + 踩过坑吗？
- 还是只是"看起来将来有用"？
- 后者一律不装。
